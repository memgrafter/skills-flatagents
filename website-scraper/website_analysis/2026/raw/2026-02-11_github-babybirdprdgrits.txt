Grits transforms from a passive tracker into an Active State Store (World Model) for agents. It enforces a strict separation between research and execution to minimize LLM "halucinations" and context drift.
- User Intent (
description
): The high-level goal set by a human or orchestrator. - Planner Agent: Analyzes the intent, researches the code via
gr star
, and populates the Implementation Plan (design
). - Coder Agent: Wakes up to a Rich Pulse JSON. If
design
is present, it begins coding. It logs every iteration/rollout to the Execution Log (notes
) viagr update --append
.
gr pulse
: Returns a full Rich Context JSON (Intent + Plan + Rules + History).gr update --append
: Persistent append-only logging for the Lab Notebook (notes
).gr workon
: Automatically warns if a Coder starts work without adesign
.
Hydrate your session (or your agent's context) with the current project status.
gr pulse
gr create "Fix login bug" --description "intent" --design "plan" --start-work
Link existing tasks or migrate them into a hierarchy.
gr dep add gr-child123 gr-parentabc --migrate
# ✅ Result: Child is renamed to gr-parentabc.n
Bundle relevant code even if you don't have a topology map yet.
gr context assemble --symbols "README.md,src/main.rs"

- Planner Skill: .agent/skills/grits-plan/SKILL.md — Self-contained Architect instructions.
- Coder Skill: .agent/skills/grits-code/SKILL.md — Self-contained Builder instructions.
- Workflows: .agent/workflows/ — Automated turbo-execution paths.
- Standard: Follows the Agent Skills open specification.
Twin Engine:
- SQLite (
.grits/grits.db
) — Fast local queries. - JSONL (
.grits/issues.jsonl
) — Git-versioned source of truth.
Graph-Lite:
- Uses a lightweight AST parser to build a dependency graph.
- Supports Blind Assembly fallback for non-topological file access.

### TL;DR
devenv is a Nix-based developer environment tool that provides fast (<100ms cached), declarative, reproducible development shells with built-in process management, container generation, and 100,000+ packages across 50+ languages.

### Key Quote
"Activate your environment in under 100ms with precise Nix evaluation caching."

### Summary

**What it is**: A developer environment manager built on Nix that replaces ad-hoc setup scripts with declarative configuration. Current version: 1.0.3.

**Core configuration**: Uses `devenv.nix` files with a module system pattern:
```nix
{ pkgs, config, ... }: {
  env.GREET = "determinism";
  packages = [ pkgs.ncdu ];
  enterShell = ''echo hello ${config.env.GREET}'';
}
```

**Key commands**:
- `devenv shell` — Enter the development environment
- `devenv up` — Start processes (like `foreman start`)
- `devenv test` — Run tests with all processes active
- `devenv container build/run` — Containerize environments
- `devenv search <term>` — Search packages and options

**Standout features**:
- **Speed**: 4.8s cold start, 47ms cached via automatic file tracking
- **Languages**: 50+ with built-in tooling (LSPs, formatters, linters), version support for Python, Rust, Go, PHP, Ruby, Terraform
- **Services**: Pre-configured PostgreSQL, Redis, MySQL, RabbitMQ, ElasticSearch, etc.
- **Processes**: Procfile-style declarative process definitions via process-compose
- **Git hooks**: Integrates git-hooks.nix for linters/formatters
- **Profiles**: Switch between frontend/backend/fullstack configurations
- **Monorepo support**: Import and merge environments across folders
- **Ad-hoc environments**: `devenv -O languages.python.enable:bool true shell` — no config files needed
- **Containers**: Generate shell/processes containers directly from dev environment
- **Secrets**: SecretSpec integration (Keychain, 1Password, LastPass, dotenv)
- **direnv integration**: Automatic environment activation on directory entry

**Architecture**: No daemons required. Works on Linux, macOS (x64/ARM64), and WSL2.

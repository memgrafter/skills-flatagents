### TL;DR
A Python framework for defining LLM agents and state machines purely in YAML, designed to be easily read and generated by LLMs themselves.

### Key Quote
"Use machines to write flatagents and flatmachines, they are designed for LLMs."

### Summary
**What it is**: A declarative orchestration framework where agents and workflows are YAML configs, not code. Inspired by Kubernetes manifests.

**Core components**:
- **FlatAgent**: Single LLM call — model + system prompt + user template + output schema
- **FlatMachine**: State machine orchestrating agents, actions, and sub-machines with branching, loops, retry, error recovery
- **Profiles**: Centralized model configs (e.g., `fast-cheap`, `smart-expensive`) referenced by name

**Install**: `pip install flatagents[all]`

**Basic usage**:
```python
from flatagents import FlatAgent
agent = FlatAgent(config_file="reviewer.yml")
result = await agent.call(query="Review this code...")
```

**Agent YAML structure**:
- `spec`/`spec_version`: Format identifier (currently "1.1.1")
- `data.model`: Profile name, inline config, or profile with overrides
- `data.system`/`data.user`: Prompts (user uses Jinja2 templating with `{{ input.* }}`)
- `data.output`: Structured schema (str, int, float, bool, json, list, object, enum)

**Execution modes**: `default`, `retry` (with backoff/jitter), `parallel` (n samples), `mdap_voting` (consensus)

**Hook system**: Extend machines via `MachineHooks` class — `on_state_enter`, `on_transition`, `on_action`, `on_error`, etc.

**Backends**: LiteLLM (default) or AISuite via `pip install flatagents[litellm]`

**Distributed workers**: Built-in `DistributedWorkerHooks`, `RegistrationBackend`, `WorkBackend` for worker pools with stale worker reaping

**Observability**: OpenTelemetry metrics (configurable via env vars), structured logging

**Examples included**: helloworld, writer_critic, story_writer, human-in-the-loop, error_handling, dynamic_agent, character_card, mdap, gepa_self_optimizer, research_paper_analysis, multi_paper_synthesizer, support_triage_json, distributed_worker, parallelism

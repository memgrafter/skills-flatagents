## What's next: `start` command

The immediate next step is adding a `start` command to the skill so you can execute a machine from the registry.

### Minimum viable `start`

```bash
run.sh start --name tagline-writer --input '{"task": "write a tagline"}'
```

### What it needs to do

1. Load config from registry (`registry.get_latest(name).config_raw`)
2. Write to temp file or pass directly to `FlatMachine(config_string=...)`
3. Resolve model profiles (`--profiles <path>` or `FLATMACHINE_PROFILES` env var)
4. Instantiate `FlatMachine` with config + profiles
5. Call `await machine.execute(input=json.loads(input_arg))`
6. Print output

### The decision points

1. **Where does `profiles.yml` live?** Options: env var, flag, `~/.agents/machine-manager/profiles.yml`, or bundled in registry alongside the machine config.

2. **Hooks for template-generated machines** — default no-op hooks work fine. No custom hooks needed. This covers all 6 templates.

3. **Hooks for the manager machine** — needs `ManagerHooks` which require the registry. This is a special case. Could be handled by the entrypoint metadata in the plan (Phase 1).

4. **The `--entrypoint` flag** — depends on the spec addition from the plan. Skip for v0, add when entrypoints land.
